# 文档维护工作流

## 每次修复/更新时必须同步的文档

### 1. updatedocs/CHANGELOG.md（详细版本日志）
- 添加新版本号和日期
- 详细列出所有变更
- 分类：🐛修复、✨新功能、🛠️改进、📝文档

### 2. map-app/CHANGELOG.md（简要版本日志）
- 同步更新最新版本摘要
- 保持简洁（3-5行）

### 3. updatedocs/DEVELOPER_GUIDE.md（开发者指南）
- 如有新模块/函数，添加说明
- 更新项目结构
- 更新调试技巧

### 4. updatedocs/API_REFERENCE.md（API参考）
- 如有新公开函数，添加文档
- 更新全局变量列表

---

## 版本号规范

`主版本.次版本.修订号`

- **主版本**：重大架构变更
- **次版本**：新功能添加
- **修订号**：Bug 修复

---

## 文档模板

### CHANGELOG 条目模板

```markdown
## [vX.X.X] - YYYY-MM-DD

### 🐛 修复
- 修复内容描述

### ✨ 新功能
- 功能描述

### 🛠️ 改进
- 改进描述

### 📝 文件变更
- `filename.js` - 变更描述（~行数）
```

---

## 更新步骤

1. 完成代码修改
2. 更新 updatedocs/CHANGELOG.md（详细）
3. 更新 map-app/CHANGELOG.md（简要）
4. 必要时更新其他文档
5. 测试验证
6. 提交代码
